Email Activity Table — Layout Proposal

Comparing current 5-column layout vs. proposed 3-column compact layout

4 changes to save horizontal space and give more room to descriptions:
  1. Merge Date + Professional into one stacked cell — date on top, employee name below, clock in/out times on the third line
  2. Add clock in/out times under the employee name in small muted text (e.g., "8:00 AM – 2:30 PM")
  3. Move Service badge inline into the Description column — appears as a colored tag before the text
  4. Eliminate 2 columns — goes from 5 columns to 3 columns (Date/Pro, Description, Hours)

The Description column grows from ~40% usable width to ~65%, giving notes and service details significantly more breathing room on narrow screens and in Outlook's rendering engine. Clock times are included without adding any column width.

5 columns — Description gets ~40% of available width
Change 1 — Merge "Date" + "Professional" into one stacked cell.
The date appears on top in normal weight. The employee name appears below in smaller, gray text. This eliminates one full column and keeps both pieces of info scannable at a glance. When the same employee has multiple entries on the same date, the repetition is visually quieter.
Change 2 — Add clock in/out times under the employee name.
The time entries already carry startTime and endTime fields from QB Workforce (clock in/out). These now appear as a third line in the stacked cell in small muted text: "8:00 AM – 2:30 PM". This gives customers visibility into when work was performed without adding any column. If times aren't available (e.g., manual entries), the line is simply omitted.
Change 3 — Move "Service" badge inline into Description.
The colored service category tag (Site Inspection, Report Writing, etc.) becomes a prefix inside the Description cell instead of occupying its own column. This eliminates a second column. The badge still provides quick visual scanning via its color — it just lives at the start of the description text rather than in a separate narrow column.
Change 4 — Result: 3 columns instead of 5.
The Description column grows from ~40% to ~65% of the table width. Notes, service details, and scope descriptions no longer get squeezed — especially on Outlook desktop where the rendering engine is tight. The Hours column stays right-aligned and bold on the far right, easy to scan down.

Change 5 — NEW/UPD badges move next to the date.
