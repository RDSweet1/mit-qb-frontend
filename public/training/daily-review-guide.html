1. Overview

▶

What is the Daily Financial Review?

The Daily Financial Review helps you track and categorize every financial transaction that flows through QuickBooks. Its purpose is to ensure nothing slips through the cracks, categorize expenses accurately, and identify unusual or unexpected transactions before they become problems.

How transactions get here

Transactions are synced from QuickBooks Online automatically. The system pulls the following transaction types:

  • Purchases — credit card charges and direct debits
  • Bills — vendor invoices
  • BillPayments — payments made to vendors
  • Transfers — bank-to-bank transfers
  • VendorCredits — credits received from vendors
  • Payments — customer payments received
  • Deposits — bank deposits

What you see for each transaction

Each transaction displays the vendor or customer name, dollar amount, date, line items, and its current categorization status. This gives you a complete picture of every financial event so you can verify, categorize, and approve it.

2. Navigating to Daily Review

▶

How to get there

  1. Sign in to the app at https://rdsweet1.github.io/mit-qb-frontend/ using your MIT Consulting Microsoft 365 account
  2. Click "Daily Review" in the top navigation bar
  3. The page loads showing today's transactions by default
Access The Daily Review tab is available to all users with view access. No special permissions are required to browse transactions.

Page layout

When the page loads, you will see:

  • Date filter at the top — select the date range you want to review
  • Sync button — pulls the latest transactions from QuickBooks Online
  • Transaction table — lists all transactions for the selected date range with status indicators

3. Syncing Transactions

▶

Manual sync

  1. Click the "Sync Transactions" button at the top of the page
  2. The system connects to QuickBooks Online and pulls the latest data
  3. New transactions appear in the table once the sync completes

What gets synced

The sync pulls all supported transaction types from QuickBooks Online:

Transaction TypeDescription
PurchaseCredit card charges, direct debits
BillVendor invoices
BillPaymentPayments to vendors
TransferBank transfers
VendorCreditCredits from vendors
PaymentCustomer payments received
DepositBank deposits
Sync duration Sync may take 30–60 seconds for large date ranges. The button will show a loading indicator while the sync is in progress.

Automatic sync

In addition to manual syncing, the system runs an automatic sync on a schedule. Check the Settings page to see the current sync timing and adjust it if needed.

4. Reviewing Transactions

▶

Transaction details

Each transaction card displays the following information:

  • Date — when the transaction occurred
  • Vendor/Customer name — who the transaction is with
  • Amount — the dollar value
  • Account — the QB account associated with the transaction
  • Line items — expandable detail showing individual line items
  • Memo — any notes or descriptions from QuickBooks

Status badges

BadgeMeaning
PendingNeeds review — no one has looked at this transaction yet
ReviewedSomeone manually reviewed and approved this transaction
Auto-ApprovedMatched a categorization rule and was automatically approved
FlaggedNeeds attention — something unusual was detected

Transaction classes

Transactions are classified into two main types:

  • Expense — money going out (purchases, bills, bill payments, vendor credits)
  • Revenue — money coming in (customer payments, deposits)

How to review

  1. Look at each transaction with a Pending status
  2. Verify the vendor or customer name and amount are correct
  3. Check that the category assignment is appropriate
  4. Mark the transaction as reviewed

5. Categorizing Expenses

▶

Why categorization matters

Categories determine how expenses appear in profitability reports and overhead analysis. Accurate categorization is essential for understanding where money is going and making informed business decisions.

Category sources

Transactions can receive categories from different sources:

SourceDescription
autoMatched by an existing categorization rule
vendorUsed the vendor's default category
manualYou chose the category yourself
recurringMatched a recurring transaction rule

How to categorize

  1. Find a transaction that has no category assigned
  2. Click the category dropdown on the transaction
  3. Select the appropriate category from the list
  4. The category is saved automatically — no need to click a separate save button
Vendor rules Setting a category for a vendor creates a rule — future transactions from that same vendor will be auto-categorized using your selection. This saves time as you build up your vendor mappings.

The overhead flag

Transactions can be marked with an is_overhead flag, which identifies them as overhead expenses. Overhead-flagged transactions are included in the Overhead analysis page and factor into overall profitability calculations.

6. Marking Days Complete

▶

What does "day complete" mean?

After reviewing all transactions for a given day, you can mark the day as complete. This provides a visual indicator that every transaction for that date has been reviewed and categorized, helping you track your progress across the week.

How to mark a day complete

  1. Review all transactions for the selected date
  2. Verify that all transactions have correct categories assigned
  3. Check the completion indicator to mark the day as done
Optional but recommended Marking a day complete is optional but helps track review progress. It gives you and other team members a clear view of which days have been fully reviewed.

7. FAQ

▶

What happens if new transactions appear after I've reviewed a day?

New synced transactions will appear as Pending regardless of the day's completion status. You'll need to review the new transactions individually, even if the day was previously marked complete.

Can I change a category after marking a transaction as reviewed?

Yes, categories can always be updated. The transaction's review status is independent of its categorization. Changing a category does not affect the reviewed/pending status.

What's the difference between "Reviewed" and "Auto-Approved"?

Reviewed means someone manually checked the transaction and marked it as reviewed. Auto-Approved means the transaction matched a categorization rule and was automatically approved by the system without manual intervention.

How do expenses here connect to the Overhead page?

Transactions marked as overhead (using the is_overhead flag) feed directly into the Overhead analysis page. The categories you assign here determine how those expenses are grouped and displayed in overhead reports.

What if a transaction is miscategorized?

Simply change the category using the dropdown on the transaction. The change takes effect immediately in all reports, including profitability and overhead analysis. There is no need to re-review or re-approve the transaction.
