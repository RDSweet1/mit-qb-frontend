🔒 How to Close a File

Prevent reports, follow-ups, and invoices from being sent after a project ends

⚠️ Why This Matters

The problem: When a customer file is closed but stray time entries exist in QuickBooks (even 0.3 hours), the system will automatically include them in weekly reports and send them to the customer. This can cause confusion and upset customers.

The "Close File" feature stops all outbound communications for a customer:

  • Weekly time reports (Monday automated sends)
  • Supplemental reports (after edits)
  • Follow-up reminder emails (3-day sequence)
  • Auto-accept confirmations
  • Manual report sends from the Time Entries page
  • Auto-generated invoices
Time entries still sync from QuickBooks Workforce — no data is lost. The entries are just excluded from all outbound reporting. You can always reopen a file later.

🔨 How to Close a File

  1. Go to the Invoices page
    Click the "Invoices" tab in the top navigation bar.
  2. Find the "Closed Files" section
    Below the "Customer Acceptance Gate Overrides" section, you'll see a collapsible panel labeled 🚫 Closed Files. Click to expand it.
  3. Select the customer to close
    Use the dropdown at the bottom of the section labeled "Close a file...". Select the customer whose file you want to close.
  4. Confirm it appears in the closed list
    The customer will immediately appear in the closed files list with a red background, showing the date it was closed and who closed it.

🔓 How to Reopen a File

If a file was closed by mistake, or the project resumes:

  1. Go to the Invoices page and expand Closed Files.
  2. Click the green "Reopen" button next to the customer's name. The customer is immediately removed from the closed list, and all automated reporting resumes.
Note: Reopening a file does not retroactively send any missed reports. It only resumes reporting from the next scheduled run forward.

❓ Frequently Asked Questions

Q: Will closing a file delete any time entries?

No. Time entries continue to sync from QuickBooks and are preserved in the database. They just won't be included in any outbound reports or invoices.

Q: Can I still view time entries for a closed file?

Yes. You can still filter by the customer on the Time Entries page — they'll appear with a [CLOSED] prefix. You can view all historical data normally.

Q: What if a technician accidentally logs time to a closed file?

The time entry will still sync into the system but will NOT be reported to the customer. You can see these entries on the Time Entries page and reassign or delete them as needed.

Q: Who can close and reopen files?

Any user with access to the Invoices page. The system records who closed the file and when, so there's a full audit trail.
